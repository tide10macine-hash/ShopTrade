# ShopTrade

Universal price comparison + resale finder — BrickSeek meets CamelCamelCamel meets
Honey, with a resale/flip layer on top.

Search a product by name or UPC, compare its price across a curated set of
trust-scored retailers, see the price history, and — for flippable categories
(sneakers, electronics, collectibles) — check what it tends to resell for after
marketplace fees.

## Why this scope

The hardest part of "compare every trusted store" is data sourcing, not code.
This MVP is built around that constraint on purpose:

- **A curated retailer set, not "every store."** 97 retailers to start
  (`src/data/retailers.json`) spanning general merchandise, apparel/footwear
  (sportswear, mall/fast fashion, off-price, department stores, shoe
  specialty, sneaker resale marketplaces), electronics, home/furniture,
  pharmacy/beauty, pet, books/toys, jewelry, and outdoor/sporting goods —
  each picked deliberately. Coverage grows by adding retailers to that list,
  not by scraping indiscriminately; see `/retailers` in the app for the full,
  current roster.
- **"Trusted" is computed, not claimed.** `src/lib/trust.ts` scores every retailer
  on SSL, registered business status, BBB rating, Trustpilot standing (discounted
  if the review sample is thin), and manual review. See it in the UI at
  `/retailers`.
- **Official APIs first, scraping as a last resort.** The real data-sourcing
  strategy is affiliate networks and product APIs — the same ones Honey,
  RetailMeNot, and Google Shopping run on. See "Data sourcing" below.
- **Brand-direct storefronts only carry their own brand.** Nike.com never
  shows a Lululemon offer. `retailers.json` marks single-brand stores with
  `brandOnly`, and `scripts/seed.mjs` filters (and guarantees a brand's own
  store is included for its own products) so comparisons stay realistic —
  see `retailerCarriesBrand()` in that file.
- **"View deal" is honest about not having live deep links yet.** Without a
  real per-retailer product catalog, a link to a URL we can't verify exists
  (a search-engine result, a guessed product-page URL) is worse than no link.
  Offers instead point at that retailer's own on-site search for the product
  name — a verified URL pattern for the ~18 retailers in
  `searchUrlTemplate`, a generic `{domain}/search?q=` guess otherwise —
  always a real, working page. True single-SKU deep links are what the live
  adapters below are for.

## Data sourcing

| Source | Role |
|---|---|
| Amazon Product Advertising API (PA-API) | Primary |
| Walmart Affiliate API | Primary |
| eBay Browse API (+ sold listings for resale comps) | Primary |
| Google Shopping Content API | Primary |
| Rakuten Advertising | Primary |
| CJ Affiliate | Primary |
| Skimlinks | Primary |
| Direct page scraping | Last resort — public price pages only, respect `robots.txt`, expect rate limits/bans |

This MVP ships with **deterministic seed data** (`npm run seed` regenerates it)
standing in for those live feeds, since none of them are wired up with real
credentials yet. The adapter layer in `src/lib/retailers/liveAdapters.ts` shows
exactly where each integration plugs in:

- Each adapter reports `isConfigured()` based on whether its required env vars
  are set (see the table in that file).
- Until configured, calling an adapter throws with the specific env vars and
  program you need to set up.
- The `/retailers` page reflects this live — each card shows **Live** or **Demo
  data** depending on whether that retailer's API source is configured.

To bring a retailer online: get program approval, set its env vars, implement
the real request in its adapter, and point `src/lib/products.ts` at it instead
of the seed data for that source.

## Architecture

```
src/
  types/           Domain types (Product, Retailer, TrustScore, ResaleEstimate, …)
  data/            Seed data: retailers.json (curated, hand-authored) and
                    products.json (generated by scripts/seed.mjs)
  lib/
    products.ts      Data access: search, lookups, "lowest offer", trending
    trust.ts          Computes the retailer trust score + verified badge
    resale.ts          Resale margin estimate: comps, fees, net value
    affiliate.ts       Builds outbound affiliate links from a retailer template
    retailers/
      types.ts          RetailerAdapter interface
      liveAdapters.ts    Stub adapters per API/affiliate network (see above)
  components/       SearchBar, PriceComparisonTable, PriceHistoryChart,
                    ResalePanel, TrustBadge, RetailerCard, AlertSignupForm, …
  app/
    page.tsx                 Home: search + trending price drops
    search/page.tsx          Search results
    product/[id]/page.tsx    Price comparison, price history, resale estimate
    retailers/page.tsx       Retailer network transparency page
    about/page.tsx           Roadmap
    api/search/route.ts      JSON search endpoint
    api/alerts/route.ts      Price-drop alert signup (stub)
```

## Phased roadmap

**MVP — this repo**
- Search by product name or UPC
- Price comparison across a curated, trust-scored retailer set
- Weekly price history chart per retailer
- Price-drop alert signup (delivery is stubbed — see `api/alerts/route.ts`)

**V2 — next**
- Resale/flip module: eBay sold-listing + StockX comps for flippable categories,
  net margin after marketplace + payment processing fees and shipping
  (already modeled in `src/lib/resale.ts`, currently fed by seed comps)
- Sell-through channel comparison: net proceeds across eBay, StockX, Facebook
  Marketplace (local pickup, no fee), and a self-hosted Shopify storefront —
  same assumed asking price, different take rates
  (`computeResaleChannels()` in `src/lib/resale.ts`)
- Real price-drop alerts via a background job + email provider
- Browser extension for on-page price comparison
- Live per-retailer product URLs via the adapters in `liveAdapters.ts`,
  replacing the on-site search links above with true deep links

**V3 — later**
- Personalized deal feed
- Community deal-sharing (Slickdeals-style)

## The resale line this doesn't cross

The resale panel is informational only: "here's where this tends to resell, and
roughly what you'd net after fees." It never places orders or automates bulk
buying. Sneaker-bot-style automation risks getting banned from affiliate
programs and retailer sites — not worth it for a feature that's valuable purely
as information.

## Monetization

Affiliate commissions on outbound clicks (`buildAffiliateLink` in
`src/lib/affiliate.ts`) — the proven model for every competitor in this space
(Honey, RetailMeNot, CamelCamelCamel, Slickdeals). A premium tier for advanced
alerts and resale analytics is a natural add-on once the core loop has usage.

## Getting started

```bash
npm install
npm run seed   # (re)generate src/data/products.json
npm run dev    # http://localhost:3000
```

Other scripts: `npm run build`, `npm run start`, `npm run lint`.
